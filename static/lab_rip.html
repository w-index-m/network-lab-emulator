Si-R (Router-A) に RIP を設定する
未完了

Si-RでRIPを有効化し、ネットワークを登録します。
Router-A の端末にコマンドを入力してください。

# RIPを有効化
Router-A# ip rip use use
# 配信するネットワークを登録
Router-A# ip rip network 192.168.1.0/24
Router-A# ip rip network 10.0.0.0/30
💡 Si-RのRIPコマンドは「ip rip use use」から始まります（Ciscoと異なります）
2
Catalyst (Dist-SW) に RIP を設定する
未完了

CatalystでRIPを有効化します。
Cisco形式のコマンドで設定してください。

Dist-SW# router rip
Dist-SW(config-router)# version 2
Dist-SW(config-router)# network 192.168.2.0
Dist-SW(config-router)# network 10.0.0.0
💡 CiscoのRIPはClassfulなので /24 などマスクを書かずネットワークアドレスだけ書きます

RIPネイバーが確立したことを確認する
未完了

両ルーターでRIPネイバーを確認します。
学習元のIPアドレスと最終更新時刻が表示されれば成功です。

Router-A# show ip rip neighbor
Dist-SW# show ip rip neighbor
期待される出力例: Index IP Address Last Update Bad Pkts Bad Routes 1 10.0.0.2 00:00:15 0 0
4
RIPルーティングテーブルを確認する
未完了

「*R」が付いた行がRIPで学習した経路です。
メトリック（ホップ数）も確認してください。

Router-A# show ip rip route
Dist-SW# show ip rip route
期待される出力（*R = RIP学習経路）: FP Destination/Mask Gateway Metric Time Interface *C 192.168.1.0/24 1 none lan0 *R 192.168.2.0/24 Dist-SW 2 00:00 lan0
💡 メトリック=2 は「1ホップ先から受信してカウントアップ」したホップ数です
5
show running-config で設定を確認する
未完了

投入したRIP設定がrunning-configに反映されているか確認します。

Router-A# show run
Dist-SW# show run
Dist-SW の期待される出力（抜粋）: router rip version 2 network 192.168.2.0 network 10.0.0.0
6
ping で疎通確認する
未完了

RIPで経路を学習した後、ping で実際に通信できるか確認します。

Router-A# ping 192.168.2.1
期待される出力: Sending 5, 100-byte ICMP Echos to 192.168.2.1, timeout is 2 seconds: !!!!! Success rate is 100 percent (5/5)
💡「!!!!!」が5個表示されれば疎通成功！「.....」は失敗（経路なし）

📚 学習ポイント

  • RIPはホップ数をメトリックとする距離ベクトル型プロトコル
  • 最大ホップ数は15（16=到達不能）
  • 30秒ごとにルーティングテーブル全体を広告（Update）
  • Si-Rは「ip rip use use」、Ciscoは「router rip」で有効化
  • 「*R」= RIP学習経路、「*C」= 直接接続（Connected）
  • AD値（Administrative Distance）= 120
  • RIPv2 はクラスレスルーティング対応（サブネットマスクを含む）
